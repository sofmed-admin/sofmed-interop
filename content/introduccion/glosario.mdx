# Glosario

## Términos del sistema de salud colombiano

| Término | Definición |
|---------|-----------|
| **EPS** | Entidad Promotora de Salud. Aseguradora que afilia usuarios y coordina la prestación de servicios. |
| **IPS** | Institución Prestadora de Servicios de Salud. Hospital, clínica o centro que presta atención directa. |
| **Referencia** | Envío de un paciente de una IPS a otra para recibir atención que la primera no puede proveer. |
| **Contrareferencia** | Retorno del paciente a su IPS de origen con el resumen de la atención recibida. |
| **CUPS** | Clasificación Única de Procedimientos en Salud. Codificación colombiana de procedimientos médicos. |
| **CIE-10** | Clasificación Internacional de Enfermedades, 10ª revisión. Usada para codificar diagnósticos. |
| **RIPS** | Registro Individual de Prestación de Servicios de Salud. |
| **MIPRES** | Mi Prescripción. Plataforma para prescribir tecnologías en salud no cubiertas por el PBS. |
| **PBS** | Plan de Beneficios en Salud. Conjunto de servicios cubiertos por el sistema. |
| **SNS** | Superintendencia Nacional de Salud. Ente de vigilancia y control. |

## Términos FHIR

| Término | Definición |
|---------|-----------|
| **FHIR** | Fast Healthcare Interoperability Resources. Estándar de interoperabilidad de HL7. |
| **R4** | Release 4. Versión normativa estable de FHIR. |
| **Recurso (Resource)** | Unidad básica de información en FHIR (ej: Patient, Encounter, ServiceRequest). |
| **Perfil (Profile)** | Restricción o extensión de un recurso FHIR para un contexto específico. |
| **Extensión (Extension)** | Campo adicional que se agrega a un recurso FHIR para datos no contemplados en el estándar base. |
| **Bundle** | Colección de recursos FHIR agrupados en una sola transacción o mensaje. |
| **CodeSystem** | Vocabulario controlado que define un conjunto de códigos (ej: tipos de documento de identidad). |
| **ValueSet** | Subconjunto de códigos de uno o más CodeSystems, usado para validación. |
| **StructureDefinition** | Definición formal de un perfil o extensión FHIR. |
| **CapabilityStatement** | Declaración de las capacidades de un servidor FHIR. |

## Términos técnicos

| Término | Definición |
|---------|-----------|
| **REST API** | Interfaz de programación basada en HTTP para intercambio de datos. |
| **JSON** | JavaScript Object Notation. Formato ligero de intercambio de datos. |
| **OAuth 2.0** | Protocolo estándar de autorización para APIs. |
| **Endpoint** | URL específica de una API que acepta peticiones. |
| **Payload** | Cuerpo de datos enviado o recibido en una petición HTTP. |
| **Interoperabilidad** | Capacidad de dos o más sistemas de intercambiar y usar información mutuamente. |
