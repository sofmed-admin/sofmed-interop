# Checklist de Adopción para EPS

Usa esta lista para evaluar qué tan preparada está tu EPS para adoptar el estándar Sofmed Interop.

## Infraestructura técnica

- [ ] **Sistema de información con API REST** — Tu sistema puede exponer y consumir APIs REST
- [ ] **Soporte para JSON** — Tu sistema puede procesar datos en formato JSON
- [ ] **Autenticación OAuth 2.0** — Tu sistema soporta autenticación basada en tokens
- [ ] **Servidor FHIR** (opcional) — Tienes o puedes implementar un servidor FHIR
- [ ] **Conectividad** — Tienes conectividad de red con las IPS de tu red prestadora

## Capacidad organizacional

- [ ] **Equipo técnico disponible** — Tienes desarrolladores que pueden implementar la integración
- [ ] **Sponsor ejecutivo** — Hay un líder que respalda la adopción del estándar
- [ ] **Presupuesto asignado** — Hay recursos para el proyecto de implementación
- [ ] **Plan de cambio** — Has considerado cómo comunicar el cambio a tu red de IPS

## Conocimiento

- [ ] **Familiaridad con FHIR** — Tu equipo técnico tiene conocimiento básico de FHIR
- [ ] **Entendimiento del estándar** — Han revisado la [visión general](/docs/estandar/vision-general)
- [ ] **Flujos claros** — Entienden el [flujo de referencia](/docs/estandar/flujo-referencia) y [contrareferencia](/docs/estandar/flujo-contrareferencia)

## Datos

- [ ] **Catálogo de IPS actualizado** — Tienes un directorio actualizado de tu red de IPS
- [ ] **Maestro de afiliados** — Puedes verificar la afiliación de un paciente en tiempo real
- [ ] **Códigos CUPS y CIE-10** — Tus sistemas usan estas codificaciones estándar

## Pasos siguientes

1. Completa el checklist e identifica las brechas
2. Prioriza las brechas según impacto y esfuerzo
3. Comparte la [guía para desarrolladores](/docs/guia-desarrolladores) con tu equipo técnico
4. Inicia una prueba piloto con una o dos IPS de tu red
5. Contacta al equipo Sofmed para apoyo en la adopción
