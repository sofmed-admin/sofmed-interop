# Generar Contrareferencia

## ¿Cuándo generar la contrareferencia?

La contrareferencia se genera cuando:
- Se ha completado la atención del paciente referido
- El paciente puede retornar a su IPS de origen para continuidad del cuidado
- Se tienen los resultados y recomendaciones para compartir

## Información requerida

### Obligatoria
- Diagnóstico(s) final(es)
- Procedimientos realizados
- Plan de seguimiento y recomendaciones
- Fecha y hora de la atención
- Profesional que atendió

### Cuando aplique
- Medicamentos formulados
- Resultados de exámenes realizados
- Próximas citas programadas
- Señales de alarma para el paciente

## Construir el Bundle de contrareferencia

El Bundle de contrareferencia referencia el ServiceRequest original:

```json
{
  "resourceType": "Bundle",
  "type": "transaction",
  "entry": [
    {
      "resource": {
        "resourceType": "ServiceRequest",
        "id": "{id-original}",
        "status": "completed",
        "note": [
          {
            "text": "Paciente valorado por cardiología. Se confirma HTA estadio 2. Se inicia manejo con enalapril 20mg/día. Control en 1 mes."
          }
        ]
      }
    },
    {
      "resource": {
        "resourceType": "Encounter",
        "status": "finished",
        "class": {
          "system": "http://terminology.hl7.org/CodeSystem/v3-ActCode",
          "code": "AMB"
        },
        "diagnosis": [
          {
            "condition": {
              "display": "Hipertensión esencial (primaria)"
            }
          }
        ],
        "period": {
          "start": "2024-01-20T09:00:00-05:00",
          "end": "2024-01-20T09:45:00-05:00"
        }
      }
    }
  ]
}
```

## Envío

```bash
POST /fhir/Bundle
Content-Type: application/fhir+json
Authorization: Bearer {token}
```

La contrareferencia se envía a:
1. **IPS de origen** — Para continuidad del cuidado
2. **EPS** — Para cierre del ciclo y trazabilidad

## Mejores prácticas

- **Genera la contrareferencia inmediatamente** después de la atención
- **Sé específico** en las recomendaciones de seguimiento
- **Incluye señales de alarma** que la IPS de origen debe vigilar
- **Referencia el ServiceRequest original** para mantener la trazabilidad
