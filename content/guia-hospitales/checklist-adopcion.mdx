# Checklist de Adopción para Hospitales e IPS

Usa esta lista para evaluar qué tan preparado está tu hospital para adoptar el estándar.

## Infraestructura técnica

- [ ] **Historia clínica electrónica** — Tu hospital tiene HCE con capacidad de integración
- [ ] **API REST disponible** — Tu sistema puede exponer y consumir APIs REST
- [ ] **Soporte para JSON** — Tu sistema puede procesar datos en formato JSON
- [ ] **Conectividad de red** — Tienes conexión estable a internet
- [ ] **Ambiente de pruebas** — Tienes un ambiente donde probar la integración

## Procesos

- [ ] **Flujo de referencia documentado** — Tienes documentado tu proceso actual de referencia
- [ ] **Equipo de referencia identificado** — Sabes quién gestiona las referencias en tu hospital
- [ ] **Codificación estándar** — Usas códigos CUPS y CIE-10 en tu sistema
- [ ] **Directorio de especialidades** — Tienes actualizado el listado de servicios que ofertas

## Equipo

- [ ] **Equipo técnico disponible** — Tienes desarrolladores para la integración
- [ ] **Capacitación en FHIR** — Tu equipo técnico ha revisado los fundamentos de FHIR
- [ ] **Sponsor clínico** — Un líder médico respalda la adopción
- [ ] **Sponsor administrativo** — La dirección apoya el proyecto

## Datos

- [ ] **Pacientes identificados** — Manejas tipo y número de documento de identidad
- [ ] **Profesionales registrados** — Tienes el directorio de tus profesionales de salud
- [ ] **Información institucional** — NIT, código de habilitación, nivel de complejidad actualizados

## Pasos siguientes

1. Completa este checklist e identifica brechas
2. Prioriza según impacto en la operación
3. Revisa la [guía para desarrolladores](/docs/guia-desarrolladores)
4. Inicia una prueba piloto con un servicio o especialidad
5. Expande gradualmente a más servicios
