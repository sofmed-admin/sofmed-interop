# Contribuir al Estándar

## El estándar es abierto y colaborativo

Sofmed Interop es un proyecto de código abierto. Cualquier actor del sistema de salud colombiano puede contribuir a mejorar y evolucionar el estándar.

## Formas de contribuir

### 1. Reportar problemas

Si encuentras un error o inconsistencia en la documentación o el estándar:

- Abre un **issue** en [GitHub](https://github.com/sofmed-admin/sofmed-interop/issues)
- Describe el problema claramente
- Incluye ejemplos si es posible

### 2. Proponer mejoras

¿Tienes una idea para mejorar el estándar?

- Abre un **issue** con la etiqueta `enhancement`
- Describe tu propuesta y la motivación
- Incluye ejemplos de cómo funcionaría

### 3. Contribuir código o documentación

- Haz un **fork** del repositorio
- Crea una rama con tu cambio
- Envía un **Pull Request** con una descripción clara

### 4. Compartir experiencias de implementación

Si has implementado el estándar:
- Comparte tu experiencia para enriquecer la documentación
- Reporta problemas encontrados en la implementación
- Sugiere mejores prácticas basadas en tu experiencia

### 5. Participar en discusiones

- Participa en las **Discussions** de GitHub
- Comparte tu perspectiva como EPS, IPS o desarrollador
- Ayuda a responder preguntas de la comunidad

## Proceso de contribución

1. **Proponer** — Abre un issue describiendo tu propuesta
2. **Discutir** — La comunidad revisa y discute la propuesta
3. **Consenso** — Se busca consenso entre los stakeholders
4. **Implementar** — Se implementa el cambio acordado
5. **Revisar** — Se revisa y aprueba mediante Pull Request
6. **Publicar** — Se incorpora al estándar en la siguiente versión

## Código de conducta

- Sé respetuoso y profesional
- Enfócate en los argumentos técnicos
- Reconoce las contribuciones de otros
- Busca el consenso antes de la imposición
